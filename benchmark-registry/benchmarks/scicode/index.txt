### Modeling Molecular Binding Affinity: A Gradient Descent Approach

In computational chemistry, predicting the binding affinity of a ligand to a target protein is crucial for drug discovery. One simplified model for this interaction can be represented by a linear relationship with added stochastic noise. The binding affinity ($Y$) can be thought of as a function of several molecular descriptors ($X_i$), such as lipophilicity, molecular weight, or hydrogen bond donors.

Mathematically, this relationship can be expressed as:

$Y = w_0 + w_1 X_1 + w_2 X_2 + ... + w_n X_n + \epsilon$

where:
*   $Y$ is the predicted binding affinity.
*   $w_0$ is a bias term (intercept).
*   $w_1, w_2, ..., w_n$ are weights associated with each molecular descriptor $X_1, X_2, ..., X_n$.
*   $X_1, X_2, ..., X_n$ are the independent molecular descriptor variables.
*   $\epsilon$ represents random noise, often modeled as Gaussian noise, which accounts for unmodeled factors and experimental variability.

The core challenge is to determine the optimal values for the weights ($w_i$) and the bias ($w_0$) that best describe the relationship between the molecular descriptors and the experimentally measured binding affinities. This process is known as **training** a model.

**Training** involves iteratively adjusting the weights and bias to minimize the difference between the model's predictions and the actual experimental binding affinities. A common technique for this is **gradient descent**, where we calculate the direction of steepest descent of a **loss function** (which quantifies the error) and update the weights and bias accordingly.

### Your Task

You are tasked with implementing a linear regression model to predict molecular binding affinity using a dataset of molecular descriptors and their corresponding experimentally determined binding affinities. You will need to implement the gradient descent algorithm from scratch using NumPy to train your model.

**Specifically, you need to:**

1.  **Generate Synthetic Data:** Create a synthetic dataset representing molecular descriptors and binding affinities. The synthetic data should follow the linear relationship described above, with added Gaussian noise. This will allow you to test your implementation on data with known "true" parameters.
2.  **Implement Linear Regression with Gradient Descent:**
    *   Initialize the weights ($w_i$) and bias ($w_0$) randomly. Ensure reproducibility by setting a random seed.
    *   Define a **loss function**: Mean Squared Error (MSE) is a standard choice for regression problems.
    *   Implement the **forward pass**: Calculate the predicted binding affinity given the input descriptors and current weights/bias.
    *   Implement the **loss calculation**: Compute the MSE between predictions and true binding affinities.
    *   Implement the **gradient calculation**: Derive and compute the gradients of the MSE loss with respect to each weight and the bias.
    *   Implement the **parameter update**: Adjust the weights and bias using the calculated gradients and a predefined learning rate.
3.  **Train the Model:** Run the gradient descent algorithm for a specified number of **epochs** (iterations over the entire dataset).
4.  **Evaluate the Model:** After training, evaluate the performance of your model by calculating the final MSE on the training data. Compare the learned weights and bias to the true parameters used to generate the synthetic data.
5.  **Visualize Results:** Plot the predicted binding affinities against the true binding affinities to visually assess the model's fit. Also, plot the loss function over epochs to observe the convergence of the training process.

### Input

*   **Number of samples (N):** An integer representing the number of data points in the dataset.
*   **Number of features (D):** An integer representing the number of molecular descriptors per sample.
*   **True weights ($w_{true}$):** A NumPy array of shape (D,) containing the ground truth weights.
*   **True bias ($b_{true}$):** A float representing the ground truth bias.
*   **Noise standard deviation ($\sigma$):** A float representing the standard deviation of the Gaussian noise.
*   **Learning rate ($\alpha$):** A float controlling the step size during gradient descent.
*   **Number of epochs:** An integer specifying the total number of training iterations.
*   **Random seed:** An integer for reproducible random number generation.

### Output

*   **Learned weights ($w_{learned}$):** A NumPy array of shape (D,) representing the trained weights.
*   **Learned bias ($b_{learned}$):** A float representing the trained bias.
*   **Training loss history:** A NumPy array containing the MSE loss for each epoch.
*   **Plots:**
    *   A scatter plot of true binding affinities vs. predicted binding affinities.
    *   A line plot of the training loss history over epochs.

### Constraints

*   You must use NumPy for all numerical computations, including data generation, gradient descent implementation, and calculations.
*   Do not use any automatic differentiation libraries (like PyTorch or TensorFlow). You must compute gradients manually.
*   The random seed must be set at the beginning of the script for reproducibility.

```python
import numpy as np
import matplotlib.pyplot as plt

# --- Configuration ---
RANDOM_SEED = 42
N_SAMPLES = 1000       # Number of data points
N_FEATURES = 5         # Number of molecular descriptors
TRUE_WEIGHTS = np.array([1.5, -0.8, 2.1, -1.2, 0.5]) # Example true weights
TRUE_BIAS = 0.7        # Example true bias
NOISE_STD_DEV = 0.5    # Standard deviation of Gaussian noise
LEARNING_RATE = 0.01   # Learning rate for gradient descent
N_EPOCHS = 500         # Number of training epochs

# --- Data Generation ---

def generate_synthetic_data(n_samples, n_features, true_weights, true_bias, noise_std_dev, seed):
    """
    Generates synthetic data for linear regression with added Gaussian noise.

    Args:
        n_samples (int): Number of data points.
        n_features (int): Number of independent variables (features).
        true_weights (np.ndarray): Ground truth weights.
        true_bias (float): Ground truth bias.
        noise_std_dev (float): Standard deviation of the Gaussian noise.
        seed (int): Random seed for reproducibility.

    Returns:
        tuple: (X, y) where X is the feature matrix (n_samples, n_features)
               and y is the target vector (n_samples,).
    """
    np.random.seed(seed)

    # Generate random features (molecular descriptors)
    # Assuming features are within a reasonable range, e.g., [-5, 5]
    X = np.random.uniform(-5, 5, size=(n_samples, n_features))

    # Calculate the true underlying linear relationship
    true_y = np.dot(X, true_weights) + true_bias

    # Add Gaussian noise
    noise = np.random.normal(0, noise_std_dev, size=n_samples)
    y = true_y + noise

    return X, y

# Generate the synthetic dataset
X_train, y_train = generate_synthetic_data(
    N_SAMPLES, N_FEATURES, TRUE_WEIGHTS, TRUE_BIAS, NOISE_STD_DEV, RANDOM_SEED
)

print(f"Generated {N_SAMPLES} samples with {N_FEATURES} features.")
print(f"Shape of X_train: {X_train.shape}")
print(f"Shape of y_train: {y_train.shape}")

# --- Model Implementation (Linear Regression with Gradient Descent) ---

class LinearRegressionNumpy:
    """
    A simple Linear Regression model implemented using NumPy and gradient descent.
    """
    def __init__(self, n_features, learning_rate=0.01, seed=42):
        """
        Initializes the Linear Regression model.

        Args:
            n_features (int): The number of input features.
            learning_rate (float): The step size for gradient descent.
            seed (int): Random seed for parameter initialization.
        """
        self.n_features = n_features
        self.learning_rate = learning_rate
        self.seed = seed
        self.weights = None
        self.bias = None
        self.loss_history = []

        self._initialize_parameters()

    def _initialize_parameters(self):
        """
        Initializes weights and bias with small random values.
        Ensures reproducibility by using the provided seed.
        """
        np.random.seed(self.seed)
        # Initialize weights with small random values (e.g., from a normal distribution)
        self.weights = np.random.randn(self.n_features) * 0.01
        # Initialize bias to zero or a small random value
        self.bias = 0.0 # np.random.randn(1)[0] * 0.01

    def predict(self, X):
        """
        Computes the model's predictions (forward pass).

        Args:
            X (np.ndarray): Input feature matrix of shape (n_samples, n_features).

        Returns:
            np.ndarray: Predicted target values of shape (n_samples,).
        """
        # Linear combination of features and weights, plus bias
        # y_pred = X @ w + b
        y_pred = np.dot(X, self.weights) + self.bias
        return y_pred

    def compute_loss(self, y_true, y_pred):
        """
        Computes the Mean Squared Error (MSE) loss.

        Args:
            y_true (np.ndarray): Ground truth target values (n_samples,).
            y_pred (np.ndarray): Predicted target values (n_samples,).

        Returns:
            float: The MSE loss.
        """
        m = y_true.shape[0] # Number of samples
        loss = np.mean((y_pred - y_true)**2)
        return loss

    def compute_gradients(self, X, y_true, y_pred):
        """
        Computes the gradients of the MSE loss with respect to weights and bias.

        Args:
            X (np.ndarray): Input feature matrix (n_samples, n_features).
            y_true (np.ndarray): Ground truth target values (n_samples,).
            y_pred (np.ndarray): Predicted target values (n_samples,).

        Returns:
            tuple: (dw, db) where dw is the gradient for weights (n_features,)
                   and db is the gradient for bias (scalar).
        """
        m = y_true.shape[0] # Number of samples

        # Gradient of MSE with respect to weights (dw):
        # d(MSE)/dw_i = (2/m) * sum((y_pred - y_true) * X_i)
        dw = (2/m) * np.dot(X.T, (y_pred - y_true))

        # Gradient of MSE with respect to bias (db):
        # d(MSE)/db = (2/m) * sum(y_pred - y_true)
        db = (2/m) * np.sum(y_pred - y_true)

        return dw, db

    def update_parameters(self, dw, db):
        """
        Updates the weights and bias using the computed gradients and learning rate.

        Args:
            dw (np.ndarray): Gradient for weights.
            db (float): Gradient for bias.
        """
        # Update weights: w = w - learning_rate * dw
        self.weights -= self.learning_rate * dw
        # Update bias: b = b - learning_rate * db
        self.bias -= self.learning_rate * db

    def train(self, X, y_true, n_epochs):
        """
        Trains the linear regression model using gradient descent.

        Args:
            X (np.ndarray): Training feature matrix (n_samples, n_features).
            y_true (np.ndarray): Training target vector (n_samples,).
            n_epochs (int): Number of training epochs.
        """
        self.loss_history = []
        print(f"Starting training for {n_epochs} epochs...")

        for epoch in range(n_epochs):
            # 1. Compute model's predictions (forward pass)
            y_pred = self.predict(X)

            # 2. Compute the loss
            loss = self.compute_loss(y_true, y_pred)
            self.loss_history.append(loss)

            # 3. Compute the gradients
            dw, db = self.compute_gradients(X, y_true, y_pred)

            # 4. Update the parameters
            self.update_parameters(dw, db)

            # Print loss every N epochs to monitor progress
            if (epoch + 1) % 100 == 0 or epoch == 0:
                print(f"Epoch {epoch+1}/{n_epochs}, Loss: {loss:.6f}")

        print("Training finished.")

# --- Model Training and Evaluation ---

# Instantiate the model
model = LinearRegressionNumpy(n_features=N_FEATURES, learning_rate=LEARNING_RATE, seed=RANDOM_SEED)

# Train the model
model.train(X_train, y_train, n_epochs=N_EPOCHS)

# Get learned parameters
learned_weights = model.weights
learned_bias = model.bias
training_loss_history = np.array(model.loss_history)

print("\n--- Training Results ---")
print(f"True Weights: {TRUE_WEIGHTS}")
print(f"Learned Weights: {learned_weights}")
print(f"True Bias: {TRUE_BIAS}")
print(f"Learned Bias: {learned_bias}")

# Evaluate the model on the training data after training
final_predictions = model.predict(X_train)
final_loss = model.compute_loss(y_train, final_predictions)
print(f"\nFinal MSE on training data: {final_loss:.6f}")

# --- Visualization ---

print("\nGenerating plots...")

# Plot 1: Predicted vs. True Binding Affinity
plt.figure(figsize=(10, 6))
plt.scatter(y_train, final_predictions, alpha=0.5, label='Data Points')
plt.plot([min(y_train), max(y_train)], [min(y_train), max(y_train)], 'r--', lw=2, label='Ideal Fit')
plt.xlabel("True Binding Affinity")
plt.ylabel("Predicted Binding Affinity")
plt.title("True vs. Predicted Binding Affinity")
plt.legend()
plt.grid(True)
plt.show()

# Plot 2: Training Loss History
plt.figure(figsize=(10, 6))
plt.plot(range(1, N_EPOCHS + 1), training_loss_history, marker='o', linestyle='-', markersize=3)
plt.xlabel("Epoch")
plt.ylabel("Mean Squared Error (Loss)")
plt.title("Training Loss Over Epochs")
plt.grid(True)
plt.show()

print("Plots generated successfully.")

### Simulating and Analyzing Molecular Vibrations in a Simplified Harmonic Oscillator Model

**Background:**

In computational chemistry and molecular physics, understanding the vibrational modes of molecules is crucial. These vibrations govern a molecule's thermodynamic properties, its interaction with light (spectroscopy), and its reactivity. A fundamental approach to modeling molecular vibrations is to treat the molecule as a system of masses (atoms) connected by springs (chemical bonds). For small displacements from equilibrium, the potential energy of such a system can be approximated by a harmonic potential, leading to a set of independent harmonic oscillators.

The motion of a single one-dimensional harmonic oscillator is described by the equation of motion:

$m \frac{d^2x}{dt^2} = -kx$

where:
*   $m$ is the mass of the particle.
*   $x$ is the displacement from the equilibrium position.
*   $k$ is the spring constant (representing the strength of the bond).
*   $t$ is time.

The analytical solution to this equation is $x(t) = A \cos(\omega t + \phi)$, where $\omega = \sqrt{k/m}$ is the angular frequency of oscillation, $A$ is the amplitude, and $\phi$ is the phase.

In a more realistic multi-dimensional scenario, especially for larger molecules, the system of coupled oscillators can be decoupled into a set of independent harmonic oscillators by transforming to normal modes. Each normal mode represents a specific collective vibration of the atoms. For each normal mode, there is a characteristic vibrational frequency.

import numpy as np
import matplotlib.pyplot as plt
from scipy.fft import fft, fftfreq
from scipy.optimize import curve_fit
import random

# --- Simulation Parameters ---
# Set random seed for reproducibility
random.seed(42)
np.random.seed(42)

def simulate_harmonic_oscillators(num_oscillators, masses=None, spring_constants=None,
                                  simulation_time=10.0, time_step=0.01):
    """
    Simulates the motion of N independent harmonic oscillators.

    Args:
        num_oscillators (int): The total number of independent harmonic oscillators.
        masses (list of float, optional): A list of N masses. If None, random masses are generated.
        spring_constants (list of float, optional): A list of N spring constants. If None, random spring constants are generated.
        simulation_time (float): The total duration of the simulation.
        time_step (float): The time step size for the simulation.

    Returns:
        tuple: A tuple containing:
            - time (np.ndarray): Array of time points.
            - displacements (np.ndarray): Array of shape (num_oscillators, num_time_points)
                                          containing the displacement of each oscillator over time.
            - theoretical_frequencies_hz (np.ndarray): Array of theoretical frequencies in Hz.
            - oscillator_params (list of dict): List of dictionaries, each containing 'mass', 'k',
                                                and 'theoretical_frequency_hz' for an oscillator.
    """

    if num_oscillators <= 0:
        raise ValueError("num_oscillators must be positive.")
    if simulation_time <= 0:
        raise ValueError("simulation_time must be positive.")
    if time_step <= 0 or time_step >= simulation_time:
        raise ValueError("time_step must be positive and less than simulation_time.")

    # Generate masses if not provided
    if masses is None:
        masses = [random.uniform(0.5, 5.0) for _ in range(num_oscillators)]
    elif len(masses) != num_oscillators:
        raise ValueError("Length of masses must match num_oscillators.")

    # Generate spring constants if not provided
    if spring_constants is None:
        spring_constants = [random.uniform(10.0, 50.0) for _ in range(num_oscillators)]
    elif len(spring_constants) != num_oscillators:
        raise ValueError("Length of spring_constants must match num_oscillators.")

    # Calculate theoretical frequencies in Hz
    # omega = sqrt(k/m)
    # f = omega / (2*pi)
    theoretical_frequencies_hz = np.sqrt(np.array(spring_constants) / np.array(masses)) / (2 * np.pi)

    # Prepare simulation parameters
    time_points = int(simulation_time / time_step)
    time = np.linspace(0, simulation_time, time_points)

    # Initialize displacements and velocities
    # Initial displacement and velocity can be random to explore different phases
    displacements = np.zeros((num_oscillators, time_points))
    velocities = np.zeros((num_oscillators, time_points))

    # Store oscillator parameters
    oscillator_params = []
    for i in range(num_oscillators):
        # Random initial displacement and velocity
        initial_displacement = random.uniform(-0.1, 0.1)
        initial_velocity = random.uniform(-0.1, 0.1)
        
        displacements[i, 0] = initial_displacement
        velocities[i, 0] = initial_velocity

        oscillator_params.append({
            'mass': masses[i],
            'k': spring_constants[i],
            'theoretical_frequency_hz': theoretical_frequencies_hz[i]
        })

    # --- Velocity Verlet Integration ---
    # This method is more stable and accurate than basic Euler for physics simulations.
    # It updates position and velocity in a staggered manner.
    
    # Pre-calculate forces (only needed for updating velocities)
    # F = -kx
    forces = -np.outer(spring_constants, displacements[:, 0])

    for t in range(time_points - 1):
        # 1. Calculate velocity at t + dt/2
        velocities[:, t + 1] = velocities[:, t] + 0.5 * (forces[:, t] / np.array(masses)) * time_step

        # 2. Calculate position at t + dt
        displacements[:, t + 1] = displacements[:, t] + velocities[:, t + 1] * time_step

        # 3. Calculate new forces at t + dt
        new_forces = -np.outer(spring_constants, displacements[:, t + 1])

        # 4. Calculate velocity at t + dt
        velocities[:, t + 1] = velocities[:, t + 1] + 0.5 * (new_forces / np.array(masses)) * time_step
        
        # Update forces for the next iteration
        forces[:, t+1] = new_forces

    return time, displacements, theoretical_frequencies_hz, oscillator_params

def analyze_spectra(time, displacements, time_step):
    """
    Computes the power spectrum for each oscillator and finds the peak frequency.

    Args:
        time (np.ndarray): Array of time points.
        displacements (np.ndarray): Array of shape (num_oscillators, num_time_points)
                                      containing the displacement of each oscillator over time.
        time_step (float): The time step size used in the simulation.

    Returns:
        tuple: A tuple containing:
            - peak_frequencies_hz (np.ndarray): Array of peak frequencies (in Hz) for each oscillator.
            - power_spectra (list of np.ndarray): List of power spectra for each oscillator.
            - frequencies (np.ndarray): Array of frequencies corresponding to the power spectra.
    """
    num_oscillators = displacements.shape[0]
    num_time_points = len(time)
    
    # Compute the FFT
    # The fft function computes the Discrete Fourier Transform.
    # fftfreq generates the corresponding frequencies for the FFT output.
    yf = fft(displacements, axis=1)
    xf = fftfreq(num_time_points, time_step)
    
    # We are interested in positive frequencies, so we take the first half of the spectrum.
    positive_freq_indices = np.where(xf >= 0)
    xf_positive = xf[positive_freq_indices]
    yf_positive = np.abs(yf[:, positive_freq_indices[0]])**2  # Power = |FFT|^2

    peak_frequencies_hz = np.zeros(num_oscillators)
    power_spectra = []

    for i in range(num_oscillators):
        spectrum = yf_positive[i]
        power_spectra.append(spectrum)
        
        # Find the index of the maximum power in the positive frequency spectrum
        peak_index = np.argmax(spectrum)
        peak_frequencies_hz[i] = xf_positive[peak_index]

    return peak_frequencies_hz, power_spectra, xf_positive

import re
import numpy as np

def parse_coordinate_string(coord_str, is_declination=False):
    """
    Parses a coordinate string (Declination or Right Ascension) and converts it
    to decimal degrees or hours.

    Args:
        coord_str (str): The coordinate string to parse (e.g., "+30°15'45.67"", "+00h30m15.678s").
        is_declination (bool): If True, parse as Declination. If False, parse as Right Ascension.

    Returns:
        float or None: The coordinate in decimal degrees (for Dec) or hours (for RA),
                       or None if parsing fails.
    """
    if not coord_str:
        return None

    # Regex to capture sign, degrees/hours, minutes, seconds
    if is_declination:
        # Declination: [+/-][DD]°[MM]'[SS].ss"
        # Example: +30°15'45.67"
        pattern = re.compile(r"([+-])(\d{1,3})°(\d{1,2})'(\d{1,2}(\.\d+)?)")
    else:
        # Right Ascension: [+/-][HH]h[MM]m[SS].sss
        # Example: +00h30m15.678s
        pattern = re.compile(r"([+-])(\d{1,2})h(\d{1,2})m(\d{1,2}(\.\d+)?)")

    match = pattern.match(coord_str.strip())
    if not match:
        return None

    sign_char, primary_unit_str, minutes_str, seconds_str = match.groups()[:4]

    try:
        sign = 1.0 if sign_char == '+' else -1.0
        primary_unit = float(primary_unit_str)
        minutes = float(minutes_str)
        seconds = float(seconds_str)

        # Conversion to decimal
        if is_declination:
            # Declination: DD + MM/60 + SS/3600
            decimal_value = sign * (primary_unit + minutes / 60.0 + seconds / 3600.0)
        else:
            # Right Ascension: HH + MM/60 + SS/3600
            decimal_value = sign * (primary_unit + minutes / 60.0 + seconds / 3600.0)

        return decimal_value
    except ValueError:
        # Handle cases where float conversion fails unexpectedly
        return None

def parse_goc_log_entry(line):
    """
    Parses a single line from the Galactic Object Catalog (GOC) log file.

    Args:
        line (str): A string representing one entry from the GOC log file.

    Returns:
        dict: A dictionary containing extracted 'object_id', 'object_name',
              'declination_deg', and 'right_ascension_hr'. Values are None if
              extraction or conversion fails for a specific component.
    """
    result = {
        'object_id': None,
        'object_name': None,
        'declination_deg': None,
        'right_ascension_hr': None
    }

    if not line or line.isspace():
        return result

    parts = line.strip().split()

    # 1. Extract Object ID
    # The Object ID is expected to be the first part, starting with a letter.
    if parts and re.match(r"^[A-Za-z]\d+", parts[0]):
        result['object_id'] = parts[0]
        # Remove the object ID from the parts for further processing
        remaining_parts = parts[1:]
    else:
        # If no valid object ID, assume the entire line is malformed for core info
        return result

    # 2. Extract Coordinates and Object Name from the remaining parts
    # Coordinates are always at the end. We need to identify them.
    # Declination: starts with + or - and contains '°'
    # Right Ascension: starts with + or - and contains 'h'
    
    # Find potential coordinate strings from the end of the line
    potential_coords = []
    for i in range(len(remaining_parts) - 1, -1, -1):
        part = remaining_parts[i]
        if (part.startswith('+') or part.startswith('-')) and ('°' in part or 'h' in part):
            potential_coords.append(part)
        else:
            # Stop searching for coordinates if we encounter a non-coordinate part
            # as they are expected to be at the very end.
            break

    # Reverse to get them in original order (Dec, RA)
    potential_coords.reverse()

    # Process coordinates and extract object name
    coord_string_parts = []
    name_parts = []
    
    # Iterate through remaining_parts to separate name from coordinates
    # Coordinates are at the end, anything before them is the name.
    # We need to be careful about how many parts are coordinates.
    
    # Determine how many coordinate parts are expected (0, 1, or 2)
    num_coord_parts = len(potential_coords)
    
    if num_coord_parts > 2:
        # This shouldn't happen based on format, but as a safeguard
        # assume only the last two are coordinates.
        num_coord_parts = 2
        potential_coords = potential_coords[-2:]
    
    # Split remaining_parts into name and coordinate parts
    # The last 'num_coord_parts' elements are coordinates.
    if num_coord_parts > 0:
        coord_string_parts = remaining_parts[-num_coord_parts:]
        name_parts = remaining_parts[:-num_coord_parts]
    else:
        name_parts = remaining_parts

    # 3. Parse Declination
    if len(coord_string_parts) > 0 and ('°' in coord_string_parts[0]):
        result['declination_deg'] = parse_coordinate_string(coord_string_parts[0], is_declination=True)
    
    # 4. Parse Right Ascension
    if len(coord_string_parts) > 1 and ('h' in coord_string_parts[1]):
        result['right_ascension_hr'] = parse_coordinate_string(coord_string_parts[1], is_declination=False)
    
    # 5. Extract Object Name
    if name_parts:
        result['object_name'] = " ".join(name_parts)
    
    # Final check: if object_id was found but no name/coords, name should be None
    if result['object_id'] and not result['object_name'] and \
       result['declination_deg'] is None and result['right_ascension_hr'] is None:
        # This means there were remaining parts but they didn't form a valid name
        # or coordinates. This scenario might indicate a malformed line.
        # Based on example "Just a note without coords", object_id would be None anyway.
        # If object_id *was* found, and nothing else was, then the "name" could be empty,
        # but we usually want None in that case.
        if len(remaining_parts) > 0 and not any(p.startswith('+') or p.startswith('-') for p in remaining_parts):
             # If there are remaining parts and none look like coordinates, they must be name parts.
             # This condition is already handled by the name_parts assignment.
             pass
        else:
             # If we had remaining parts, but they were all treated as potential coordinates
             # and not successfully parsed, or if there were no remaining parts after ID,
             # then the name is effectively empty.
             result['object_name'] = None # Ensure it's None if no name parts were found.


    return result

def simulate_stellar_evolution(initial_mass: float, stages_data: dict, time_steps_per_stage: int = 100) -> pd.DataFrame:
    """
    Simulates the evolution of a star through different evolutionary stages.

    Args:
        initial_mass (float): The initial mass of the star in solar masses.
        stages_data (dict): A dictionary defining the stellar evolutionary stages,
                            including luminosity range, temperature range, and duration factor.
        time_steps_per_stage (int): The number of time steps to discretize each evolutionary stage.

    Returns:
        pd.DataFrame: A DataFrame containing the simulation results with columns:
                      'time', 'luminosity', 'temperature', 'stage'.
    """
    simulation_data = []
    current_time = 0.0

    # Define a typical order of evolutionary stages
    # This order is crucial for the simulation flow
    stage_order = [
        "Main Sequence",
        "Red Giant",
        "Horizontal Branch", # Or directly to AGB for simplicity
        "Asymptotic Giant Branch",
        "White Dwarf"
    ]

    for stage_name in stage_order:
        if stage_name not in stages_data:
            print(f"Warning: Stage '{stage_name}' not found in stages_data. Skipping.")
            continue

        stage_info = stages_data[stage_name]
        min_lum, max_lum = stage_info["luminosity_range"]
        min_temp, max_temp = stage_info["temperature_range"]
        duration_factor = stage_info["duration_factor"]

        # Approximate time spent in this stage: T ~ M * duration_factor
        # More massive stars evolve faster on the main sequence, but spend more *total* time on MS.
        # The duration_factor is tricky. For simplicity here, we'll assume:
        # Time in stage = duration_factor * (initial_mass ** some_power)
        # For simplicity, let's use a simplified scaling:
        # For main sequence, duration is long for lower mass stars.
        # For later stages, duration is longer for higher mass stars.
        # To keep it simple and illustrative: Time ~ duration_factor * (initial_mass ^ 0.5) for MS
        # and Time ~ duration_factor / (initial_mass ^ 0.5) for later stages.
        # Let's use a simple multiplier for now.
        if stage_name == "Main Sequence":
            time_in_stage = duration_factor * (initial_mass ** -2.5) # Realistic scaling T ~ M^-2.5 for MS lifetime
        elif stage_name == "White Dwarf":
            time_in_stage = duration_factor * (initial_mass ** 0.5) # White dwarfs cool over time, longer for more massive progenitor
        else:
            time_in_stage = duration_factor * (initial_mass ** 1.0) # Intermediate stages roughly proportional

        if time_in_stage <= 0: # Ensure positive time
            time_in_stage = duration_factor # Fallback if scaling results in non-positive time

        dt = time_in_stage / time_steps_per_stage

        for i in range(time_steps_per_stage):
            # Sample luminosity and temperature within the defined ranges
            # Use log scale for luminosity as it's common in H-R diagrams
            log_lum = np.random.uniform(np.log10(min_lum), np.log10(max_lum))
            luminosity = 10**log_lum

            temperature = np.random.uniform(min_temp, max_temp)

            simulation_data.append({
                'time': current_time,
                'luminosity': luminosity,
                'temperature': temperature,
                'stage': stage_name
            })
            current_time += dt

    return pd.DataFrame(simulation_data)

## Simulating Diffusion-Limited Aggregation (DLA) for Biological Structures

**Introduction:**
In many natural phenomena, complex structures emerge from simple local interactions. One such process is Diffusion-Limited Aggregation (DLA), a model for how particles aggregate under diffusion. Imagine dust particles in the air slowly moving due to random Brownian motion. When two dust particles collide, they stick together. Over time, these small clusters grow into larger, often fractal, structures. DLA is a powerful model that can explain the formation of various natural and artificial patterns, including soot particles, mineral dendrites, and even the branching structures of certain biological organisms like lung alveoli or blood vessel networks.

This problem challenges you to simulate a 2D DLA process and analyze the resulting aggregate. You will implement a particle-walking algorithm and then quantify some basic properties of the generated structure.

**The Model:**
We will simulate DLA on a 2D grid. The process starts with a single "seed" particle at the center of the grid. Then, a stream of "walker" particles is introduced, starting from the edges of the grid and moving randomly one step at a time (up, down, left, or right).

1.  **Walker Movement:** Each walker moves randomly to an adjacent grid cell.
2.  **Aggregation:** If a walker moves into a grid cell that is already occupied by an aggregate particle (the seed or a previously aggregated walker), the walker stops moving and becomes part of the aggregate.
3.  **Boundary Condition:** If a walker moves outside the defined grid boundaries, it is considered "lost" and does not contribute to the aggregate. A new walker is then introduced.
4.  **Iteration:** This process is repeated until a specified number of walkers have successfully aggregated, forming the final DLA structure.

**Task:**
Implement a 2D DLA simulation and perform basic analysis on the resulting aggregate.

**Specific Requirements:**

1.  **DLA Simulation:**
    *   Represent the 2D grid as a 2D array (e.g., using NumPy). Initialize it with zeros, and set the center cell to 1 (representing the seed).
    *   Implement a function `simulate_dla(grid_size, num_aggregators)` that performs the DLA simulation.
        *   `grid_size`: An integer representing the width and height of the square grid.
        *   `num_aggregators`: The target number of particles (walkers) to aggregate (excluding the initial seed).
    *   The simulation should:
        *   Start walkers from random positions on the grid's perimeter.
        *   Allow walkers to move randomly in four cardinal directions.
        *   Check for aggregation: if a walker lands on an occupied cell (value > 0), it becomes part of the aggregate (mark its cell with a unique identifier for that walker, e.g., its starting index).
        *   Handle boundary conditions: if a walker moves off the grid, it's lost, and a new walker is introduced.
        *   Stop when `num_aggregators` walkers have successfully aggregated.
    *   The function should return the final aggregated grid.

2.  **Analysis:**
    *   **Radius of Gyration:** Calculate the radius of gyration ($R_g$) of the aggregate. This is a measure of the aggregate's size and compactness. It's defined as:
        $R_g = \sqrt{\frac{1}{N} \sum_{i=1}^{N} r_i^2}$
        where $N$ is the total number of aggregated particles (including the seed), and $r_i$ is the distance of the $i$-th particle from the center of mass of the aggregate. For simplicity, we can approximate the center of mass as the center of the grid.
    *   **Box-Counting Dimension (Optional but Recommended):** Estimate the fractal dimension of the aggregate using the box-counting method. This involves covering the aggregate with boxes of different sizes and counting how many boxes contain at least one aggregated particle. For a fractal, the number of boxes $N(s)$ scales with box size $s$ as $N(s) \propto s^{-D}$, where $D$ is the fractal dimension. You can implement this by:
        *   Iterating through a range of box sizes $s$.
        *   For each $s$, divide the grid into $s \times s$ boxes.
        *   Count how many of these boxes contain at least one aggregated particle.
        *   Plot $\log(N(s))$ vs. $\log(s)$ and estimate the slope, which approximates $-D$.

**Input:**
*   `grid_size`: An integer (e.g., 100).
*   `num_aggregators`: An integer (e.g., 1000).

**Output:**
*   The final 2D NumPy array representing the aggregated grid.
*   The calculated radius of gyration ($R_g$).
*   (Optional) The estimated fractal dimension $D$.

**Constraints:**
*   `grid_size` will be between 50 and 200.
*   `num_aggregators` will be between 500 and 5000.
*   The simulation should be reasonably efficient.

**Example Scenario:**
Imagine you are a computational biologist studying how complex cellular structures like vascular networks form. DLA provides a simplified model to understand the initial branching patterns. By simulating this process and analyzing the resulting structure's size and fractal properties, you can gain insights into the underlying growth mechanisms.

import numpy as np
import matplotlib.pyplot as plt
import random
from scipy.ndimage import gaussian_filter

def simulate_dla(grid_size: int, num_aggregators: int) -> np.ndarray:
    """
    Simulates Diffusion-Limited Aggregation (DLA) on a 2D grid.

    Args:
        grid_size: The width and height of the square grid.
        num_aggregators: The target number of walkers to aggregate (excluding the seed).

    Returns:
        A 2D NumPy array representing the aggregated grid.
        0: empty cell, >0: aggregated particle (unique ID for each aggregator).
    """
    # Initialize grid: 0 for empty, 1 for seed
    grid = np.zeros((grid_size, grid_size), dtype=int)
    center_x, center_y = grid_size // 2, grid_size // 2
    grid[center_x, center_y] = 1  # Seed particle

    aggregated_count = 0
    walker_id = 2  # Start unique IDs for aggregated walkers from 2

    # Directions: (dx, dy) for right, left, down, up
    directions = [(1, 0), (-1, 0), (0, 1), (0, -1)]

    while aggregated_count < num_aggregators:
        # Start walker from a random position on the perimeter
        edge = random.choice(["top", "bottom", "left", "right"])
        if edge == "top":
            start_x, start_y = random.randint(0, grid_size - 1), 0
        elif edge == "bottom":
            start_x, start_y = random.randint(0, grid_size - 1), grid_size - 1
        elif edge == "left":
            start_x, start_y = 0, random.randint(0, grid_size - 1)
        else: # right
            start_x, start_y = grid_size - 1, random.randint(0, grid_size - 1)

        walker_x, walker_y = start_x, start_y
        
        # Track walker path to avoid self-intersection with itself in a single step
        # (though less critical in standard DLA, good practice for robustness)
        path_history = [(walker_x, walker_y)] 

        # Walker movement loop
        while True:
            # Choose a random direction
            dx, dy = random.choice(directions)
            new_walker_x, new_walker_y = walker_x + dx, walker_y + dy

            # Boundary check
            if not (0 <= new_walker_x < grid_size and 0 <= new_walker_y < grid_size):
                # Walker is lost, break and start a new walker
                break 

            # Aggregation check
            if grid[new_walker_x, new_walker_y] > 0:
                # Aggregation occurred
                grid[new_walker_x, new_walker_y] = walker_id
                aggregated_count += 1
                walker_id += 1
                break # Walker aggregated, move to the next one

            # Update walker position
            walker_x, walker_y = new_walker_x, new_walker_y
            path_history.append((walker_x, walker_y)) # Add to path history

            # Optional: Prevent walker from immediately returning to its previous spot in one step
            # This can sometimes lead to more "open" structures but is not strictly part of basic DLA.
            # For simplicity, we'll allow it unless it causes issues.
            # If len(path_history) > 2 and path_history[-1] == path_history[-3]:
            #     # Walker is moving back and forth, potentially stuck.
            #     # We could implement a more complex 'random walk' strategy or break.
            #     # For now, we allow it.

            # Safety break for infinite loops (unlikely with aggregation check)
            if len(path_history) > grid_size * grid_size * 2: # Arbitrary large number
                print("Warning: Walker path too long, potential infinite loop. Breaking.")
                break
    
    return grid

def analyze_dla(grid: np.ndarray):
    """
    Analyzes the DLA aggregate to calculate radius of gyration and estimate fractal dimension.

    Args:
        grid: The 2D NumPy array representing the aggregated grid.

    Returns:
        radius_of_gyration (float): The calculated radius of gyration.
        fractal_dimension (float): The estimated fractal dimension (or None if calculation fails).
    """
    # Find coordinates of all aggregated particles (including the seed)
    agg_coords = np.argwhere(grid > 0)
    
    if agg_coords.shape[0] == 0:
        return 0.0, None

    # 1. Calculate Radius of Gyration
    # Center of mass approximation: average of coordinates
    # For DLA, the geometric center of the grid is often a good proxy for the center of mass.
    grid_size = grid.shape[0]
    center_x, center_y = grid_size // 2, grid_size // 2

    # Calculate squared distances from the grid center
    squared_distances = (agg_coords[:, 0] - center_x)**2 + (agg_coords[:, 1] - center_y)**2
    
    # Radius of Gyration formula
    radius_of_gyration = np.sqrt(np.mean(squared_distances))

    # 2. Estimate Fractal Dimension using Box Counting
    # This is an optional but recommended step.
    # We'll compute it only if there are enough points.
    fractal_dimension = None
    if agg_coords.shape[0] > 100: # Need a reasonable number of points for box counting
        try:
            # Define a range of box sizes (powers of 2 are common)
            # Max box size should be less than grid_size, min box size should be at least 1
            max_box_size = grid_size // 2
            if max_box_size < 2: max_box_size = 2
            min_box_size = 1
            
            box_sizes = []
            num_boxes_containing_particles = []

            # Iterate through box sizes
            # We want to cover a range from small boxes to larger ones.
            # Use a logarithmic scale or a set of powers of 2 for box sizes.
            # Let's use a range of powers of 2 from min_box_size up to max_box_size.
            current_box_size = min_box_size
            while current_box_size <= max_box_size:
                box_sizes.append(current_box_size)
                
                # Count particles within boxes of this size
                count = 0
                # Iterate through the grid in blocks of current_box_size
                for i in range(0, grid_size, current_box_size):
                    for j in range(0, grid_size, current_box_size):
                        # Define the current box
                        box_end_i = min(i + current_box_size, grid_size)
                        box_end_j = min(j + current_box_size, grid_size)
                        
                        # Check if any aggregated particle falls within this box
                        # Efficiently check using numpy's boolean indexing
                        # We need to check if any point (agg_coords) is within the range [i, box_end_i) and [j, box_end_j)
                        # This can be done by finding points where:
                        # i <= point_x < box_end_i  AND  j <= point_y < box_end_j
                        
                        # Get all points within the current box's grid area
                        points_in_grid_area = agg_coords[
                            (agg_coords[:, 0] >= i) & (agg_coords[:, 0] < box_end_i) &
                            (agg_coords[:, 1] >= j) & (agg_coords[:, 1] < box_end_j)
                        ]
                        
                        if points_in_grid_area.shape[0] > 0:
                            count += 1
                
                if count > 0: # Only record if there's something to count
                    num_boxes_containing_particles.append(count)
                
                # Move to the next box size (e.g., double it)
                if current_box_size == 1:
                    current_box_size = 2
                else:
                    current_box_size *= 2

            # Convert lists to numpy arrays for calculation
            box_sizes = np.array(box_sizes)
            num_boxes_containing_particles = np.array(num_boxes_containing_particles)

            # Remove entries where num_boxes is 0 if they were added (shouldn't happen with the check)
            valid_indices = num_boxes_containing_particles > 0
            box_sizes = box_sizes[valid_indices]
            num_boxes_containing_particles = num_boxes_containing_particles[valid_indices]

            if len(box_sizes) < 2: # Need at least 2 points to fit a line
                fractal_dimension = None
                print("Warning: Not enough distinct box sizes for fractal dimension estimation.")
            else:
                # Fit a line to log(N(s)) vs log(s)
                # N(s) = C * s^(-D) => log(N(s)) = log(C) - D * log(s)
                # So, slope = -D
                log_s = np.log(box_sizes)
                log_N = np.log(num_boxes_containing_particles)

                # Perform linear regression
                # coefficients = np.polyfit(log_s, log_N, 1)
                # fractal_dimension = -coefficients[0] # Slope is -D

                # More robust estimation using singular value decomposition (SVD)
                # or simply using numpy's polyfit
                
                # Using numpy's polyfit for simplicity and common usage
                slope, intercept = np.polyfit(log_s, log_N, 1)
                fractal_dimension = -slope

        except Exception as e:
            print(f"An error occurred during fractal dimension estimation: {e}")
            fractal_dimension = None

    return radius_of_gyration, fractal_dimension
